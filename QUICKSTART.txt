STEP 6: View Results
====================

CSV results (spreadsheet-friendly):
  ../output/query_length_experiment_5level/results_qlength_5level_128_llama3-8B.csv

Columns in CSV:
  - category: very_short/short/medium/long/very_long
  - num_queries: 200 per category
  - avg_token_len: average tokens in category
  - avg_embedding_time_ms: embedding latency
  - avg_retrieval_time_ms: retrieval latency
  - avg_generation_time_ms: generation latency
  - total_time_ms: end-to-end latency
  - throughput_qps: queries per second

ADDITIONAL OPTIONS
==================

Multiple nprobe values:

  for nprobe in 128 256 512; do
    python query_length_experiment.py \
      --config query_length_config_5level.yaml \
      --nprobe $nprobe \
      --llm_model llama3-8B
  done

Different LLM models:

  for model in llama3-8B llama2-13B opt-30B; do
    python query_length_experiment.py \
      --config query_length_config_5level.yaml \
      --nprobe 128 \
      --llm_model $model
  done

Custom output directory:

  python query_length_experiment.py \
    --config query_length_config_5level.yaml \
    --nprobe 128 \
    --llm_model llama3-8B \
    --output_dir /custom/path

Key observations:
  - Embedding increases with query length (linear)
  - Retrieval stable (±2%)
  - Generation stable (±1%)
  - Total latency increases ~7%
  - Throughput decreases ~7%

TROUBLESHOOTING
===============

Issue: CUDA out of memory
Solution:
  - Reduce GPU count in config
  - Reduce batch_size in config
  - Enable FP16 precision

Issue: Index file not found
Solution:
  - Verify index_path in config
  - Use --index_path to override

Issue: Model not found
Solution:
  - Check model_path in config
  - Verify model is downloaded/available

Issue: Slow performance
Solution:
  - May be normal (90 min for full 5-level)
  - Check GPU utilization: nvidia-smi
  - Check system load: top
